CyberTelemetrySimulator Architecture

Overview
- The simulator generates telemetry events for a set of device instances.
- Each telemetry event contains baseline metrics plus optional attack effects.
- Events are published to the console and to a JSONL file for downstream use.

Core Flow
1) Program entrypoint (Program.cs)
   - Loads configuration from Config/simulatorSettings.json.
   - Builds a CampaignManager for attack scheduling.
   - Creates a list of DeviceSimulator instances for different device types.
   - In a loop, generates telemetry and publishes it to multiple publishers.

Configuration
- Config/simulatorSettings.json defines tick interval, attack chance, attack duration bounds, and output path.
- Config/SimulatorSettings.cs maps JSON into a strongly-typed settings object.

Telemetry Model
- Models/TelemetryEvent.cs is the envelope for each telemetry event.
  - Timestamp, DeviceId, DeviceType, Metrics, Label (attack type), AttackId.
- Models/Metrics.cs defines all telemetry metrics, including derived values.
  - Baseline metrics are created per device; derived metrics are computed in DeviceSimulator.

Device Simulation
- Devices/DeviceSimulator.cs
  - Generates baseline metrics using the device profile ranges.
  - Adds small noise for realism.
  - If an attack is active, applies attack effects.
  - Recomputes derived metrics (failed login rate, ratios, after-hours flag).

Device Profiles
- Devices/DeviceProfile.cs defines baseline ranges per DeviceType.
- DeviceType (Models/DeviceType.cs) enumerates device categories.
- The profile ranges are used to generate normal telemetry for each device.

Attack Scheduling
- Campaigns/CampaignManager.cs manages AttackEpisode lifecycles.
  - Randomly starts attacks based on AttackChancePerTick.
  - Keeps attacks active for MinDurationSec..MaxDurationSec.
- Campaigns/AttackEpisode.cs stores the active attack state and progress.

Attack Effects
- Campaigns/AttackApplier.cs applies attack-specific effects to metrics.
  - Brute force: failed logins, login ratios, unique source IPs.
  - Port scan: unique ports, connection attempts, short durations.
  - DDoS: packet rate, new connections, IP count, traffic volume spikes.
  - Exfiltration: outgoing bytes spike, traffic ratio, after-hours flag.

Publishing
- Publishers/ConsolePublisher.cs prints JSON telemetry to the console.
- Publishers/FileJsonlPublisher.cs appends each event as a JSON line.
- Both implement ITelemetryPublisher for consistent publishing.

Output
- By default, telemetry is written to data/raw-telemetry.jsonl under the app base directory.
- Each line is a complete JSON object representing one TelemetryEvent.
