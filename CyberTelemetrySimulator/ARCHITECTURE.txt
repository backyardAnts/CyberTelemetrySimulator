CyberTelemetrySimulator — Architecture and Code Guide

1) Application Entry

Program.cs
- Application entry point.
- Loads configuration from Config/simulatorSettings.json.
- Creates the CampaignManager, DeviceSimulator instances, and publishers.
- Runs the main loop to generate telemetry, publish JSONL output, and optionally produce SOC alerts.

CyberTelemetrySimulator.csproj
- .NET project file with build configuration.

2) Configuration

Config/SimulatorSettings.cs
- Strongly-typed configuration model with defaults.
- Core knobs: tick interval, attack probability, attack durations, output path.
- Dataset modes: balanced mode and training mode.
- Time-of-day realism: business hours and baseline/attack multipliers.

Config/simulatorSettings.json
- Default runtime values that mirror SimulatorSettings.

3) Campaign Management (Attack Scheduling)

Campaigns/CampaignManager.cs
- Owns the list of active attack episodes per device.
- Starts new attacks based on mode:
  - Production mode: probabilistic start, scaled higher after-hours.
  - Balanced dataset mode: selects the attack type with the largest deficit.
  - Training dataset mode: round-robin attack types with fixed duration.
  - SOC incident chain: multi-stage scripted attack flow.
- Tracks counts per label and logs counts every 1000 events.

Campaigns/AttackEpisode.cs
- Represents a single attack window with type, mode (loud/stealth), intensity, timing, and optional incident ID.

Campaigns/AttackApplier.cs
- Applies attack effects by mutating baseline metrics during active episodes.

4) Device Simulation (Telemetry Generation)

Devices/DeviceSimulator.cs
- Generates baseline metrics using statistical distributions (Poisson, log-normal, AR(1)).
- Applies time-of-day multipliers to baseline activity:
  - Business hours increase baseline traffic.
  - Night hours lower baseline traffic.
  - Transitional hours use the neutral multiplier.
- Applies active attack effects and updates derived metrics (e.g., TrafficVolumeBytes).
- Sets TimeOfDay (0–23) and AfterHoursActivity (0/1) without changing the telemetry schema.

Devices/DeviceProfile.cs
- Per-device-type ranges for traffic, connections, CPU, and authentication rates.

5) Models (Telemetry Contracts)

Models/TelemetryEvent.cs
- Top-level telemetry envelope: timestamp, device identity, metrics, label, attack metadata.

Models/Metrics.cs
- All telemetry metrics, including derived fields and time-of-day indicators.

Models/AttackType.cs
- Supported attack categories: Normal, BruteForce, PortScan, DDoS, Exfiltration.

Models/AttackMode.cs
- Attack intensity modes: Loud, Stealth.

Models/DeviceType.cs
- Device categories used to select profiles.

Models/DeviceSecurityState.cs
- SOC state classifier enum.

Models/SecurityAlert.cs
- Schema for SOC alerts emitted in SOC mode.

6) Detection & Validation

Detection/DetectionEngine.cs
- Simple risk scoring and suspected attack classification based on metric indicators.

Validation/TelemetrySelfCheck.cs
- Basic sanity checks for generated metrics and alert consistency.

Utils/RandomDistributions.cs
- Statistical helpers (Poisson, log-normal, AR(1)) for realistic variability.

7) Publishing

Publishers/ITelemetryPublisher.cs
- Interface for telemetry output adapters.

Publishers/ConsolePublisher.cs
- Writes telemetry to stdout.

Publishers/FileJsonlPublisher.cs
- Appends telemetry events to `data/raw-telemetry.jsonl`.

Publishers/AlertJsonlPublisher.cs
- Writes SOC alerts to `data/alerts.jsonl`.
