CyberTelemetrySimulator — Project Structure Overview
1️⃣ Application Entry

Program.cs
Application entry point.
Initializes configuration, devices, campaign manager, detection engine, and publishers.
Runs the telemetry generation loop and optionally enables SOC mode + alert generation.

CyberTelemetrySimulator.csproj
.NET project definition and dependency configuration.

2️⃣ Configuration

Config/SimulatorSettings.cs
Strongly-typed configuration model for:

Tick interval

Attack probability

Attack duration bounds

Output file paths

SOC mode settings

Config/simulatorSettings.json
Default runtime configuration values.

3️⃣ Attack Campaign Management (Behavior Orchestration)

These components simulate realistic cyber attack behavior over time.

Campaigns/CampaignManager.cs
Central scheduler responsible for:

Starting and stopping attack episodes

Managing active attacks per device

Supporting optional SOC incident chains (multi-stage attacks)

Campaigns/AttackEpisode.cs
Represents a single attack window:

Attack type

Attack mode (Loud / Stealth)

Start and end timing

Optional incident correlation ID

Campaigns/AttackApplier.cs
Applies attack effects to device telemetry by mutating metrics during active attack windows.

4️⃣ Device Simulation (Telemetry Generation)

Responsible for producing realistic baseline telemetry.

Devices/DeviceSimulator.cs
Generates baseline metrics for a device and applies attack effects when active.

Devices/DeviceProfile.cs
Defines baseline metric ranges and behavioral characteristics per device type.

5️⃣ Data Models (Telemetry Contracts)

Defines all structured data exchanged within the system.

Models/TelemetryEvent.cs
Main telemetry envelope:

Timestamp

DeviceId

DeviceType

Metrics

Attack label

AttackId

Models/Metrics.cs
Defines all telemetry metrics including derived fields.

Models/AttackType.cs
Supported attack categories (BruteForce, PortScan, DDoS, DataExfiltration).

Models/AttackMode.cs
Attack intensity mode (Loud / Stealth).

Models/DeviceType.cs
Device categories (Workstation, WebServer, DatabaseServer, IoTDevice, etc.).

Models/DeviceSecurityState.cs
SOC state classification enum.

Models/SecurityAlert.cs
Schema for generated SOC alerts.

6️⃣ Detection & Analytics Layer

Implements simple attack detection logic and behavioral scoring.

Detection/DetectionEngine.cs
Performs:

Risk scoring

Behavioral anomaly evaluation

Suspected attack classification

Validation/TelemetrySelfCheck.cs
Lightweight sanity checks to validate metric consistency.

Utils/RandomDistributions.cs
Statistical helpers (Poisson, Log-normal, AR(1)) used for realistic traffic modeling.

7️⃣ Publishing Layer (Output Adapters)

Responsible for exporting telemetry and alerts.

Publishers/ITelemetryPublisher.cs
Interface abstraction for telemetry output mechanisms.

Publishers/ConsolePublisher.cs
Writes telemetry to console output.

Publishers/FileJsonlPublisher.cs
Appends telemetry events to a JSONL file for downstream processing.

Publishers/AlertJsonlPublisher.cs
Writes SOC alert events to JSONL format.
